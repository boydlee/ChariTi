Getting Started

This document describes how to get your application up-and-running after it's been delivered to you for testing by Appcelerator.

Information Required From You

  • Flickr API key, if using Flickr component
  • Text copy for each applicable component (Donate, Text)
  • Any other applicable configuration information (see Available Components)
  • The e-mail template for sharing (more information below)

Next Steps

NOTICE: These steps must be followed in order to ensure a successful application launch.
  1. Verify the correct components are being displayed
  2. Test the application thoroughly
  3. Send any comments or feedback to your Appcelerator contact
  4. Host a copy of the application configuration file on your server (more information below)
  5. Request a copy of all creatives (icon, splash screen, etc) and sign-off
  6. Request a copy of the final application package
  7. Request guidance from Appcelerator on submitting your app to the App Store

E-Mail Template Copy

The e-mail template is made up of 3 parts: the subject, the body, and the footer. The dynamic content (a link to the article being shared) is placed between the body and the footer.

Please provide the desired copy for all three. Example:

Subject:	Check out this great ChariTi content!

Body:		I found this and thought you might find it interesting:
Content:	http://www.mywebsite.com/news-article-1
			
Footer:		Sent via ChariTi

Hosting the Configuration File

In order to allow you to update the application immediately, and without the need to use the Titanium products, you can optionally host a version of the application configuration file (app.json) on your server.

If you'd like to do this, please provide us the URL for where the file will be hosted.

Example: http://www.mywebsite.com/files/mobile/app.json
